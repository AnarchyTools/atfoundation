Threading primitives¶

class Thread¶
init(_: ((Void) -> Any?))¶

Initialize and run a thread

Parameters:
  • block – block to run in a new thread, the return value of the block can be aquired with wait()
init(_: ((Void) -> Void))

Initialize and run a detached thread that does not return a value

Parameters:
  • block – block to detach to new thread
func wait() → Any?¶

Wait for the result of a non-detached thread

Returns:Value that the thread closure returned
static var threadID : UInt64¶

Fetch current thread ID, returns zero for main thread

class Barrier¶
init()¶

Initialize a new barrier

func wait(timeout: Int? = nil)¶

Wait for a signal (stop at the barrier)

Parameters:
  • timeout – optional, time in nano seconds to wait, set to nil or skip to wait forever
func signal()¶

Signal the barrier, all threads blocked on that barrier will resume execution

class Mutex¶
init(recursive: Bool = false)¶

Initialize new lock

Parameters:
  • recursive – Set to true if you want a recursive lock (same thread may recursively lock multiple times)
func lock()throws¶

Lock the mutex, may throw an error on resource exhaustion or multiple locking from the same thread when not recursive Usually you could use try! to execute this as every error this throws is either system resource exhaustion or a programming error.

func unlock()throws¶

Unlock a locked mutex, may throw errors if the mutex was not locked Usually you could use try! to execute this as every error this throws is either system resource exhaustion or a programming error.

func tryLock()throws → Bool¶

Try locking the mutex, returns false if the lock is already held somewhere. If it returns true the locking succeeded. Usually you could use try! to execute this as every error this throws is either system resource exhaustion or a programming error.

func whileLocking(_: (Void) -> Void)throws¶

Run a block with the lock held, this is the prefered method to use

Parameters:
  • cb – block to execute
class Semaphore¶
let name : String¶

Name of the semaphore

init(name: String, count: UInt32)throws¶

Initialize a semaphore

Parameters:
  • name – The name
  • count – Resource allocation limit
func wait()throws¶

Wait for a semaphore. This decrements the resource limit, if the limit reaches zero the thread will wait for a post() from another thread. Usually you could use try! to execute this as every error this throws is either system resource exhaustion or a programming error.

func tryWait()throws → Bool¶

Test if resources are available

Usually you could use try! to execute this as every error this throws is either system resource exhaustion or a programming error.

Returns:false if the semaphore resources are at zero already, true if the semaphore could be aquired
func post()¶

Release a resource to the semaphore

func whileLocking(cb: (Void) -> Void)throws¶

Execute a block aquiring resources from the semaphore and return the resources afterwards, this is the prefered function to call.

Parameters:
  • cb – block to execute
